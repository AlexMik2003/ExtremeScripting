Operations Detail

closeSession

Operation used to end a webservices session with the switch. The session details obtained from a openSession operation are used to populate the closeSessionRequest message.

NOTE: This operation is used only when the client has used the openSession operation to start a webservices session with the switch.

Input:
tns:closeSessionRequest
Output:
tns:closeSessionReply
Fault:

Input:
tns:createRequest
Output:

Fault:

delete

Operation used to delete an object from the switch. The object to delete is specified as part of the deleteRequest message.

Input:
tns:deleteRequest
Output:
tns:deleteResponse
Fault:

get

Operation used to get data for objects from the switch. The type of object to get and other options for the get operation are specified as part of the getRequest message. This operation can be used to get either a specific object or a set of objects of a type.

The data returned by this operation is contained in the getResponse message.

Input:
tns:getRequest
Output:
tns:getResponse
Fault:

keepAlive

Operation used to keep the webservices session open during periods of inactivity.

The switch can terminate webservices that are idle for long periods of time. The client can specify the idle timeout value as part of the openSession operation. But if there are periods of inactivity that are longer than this timeout, the client can keep the session by sending this keepAliveRequest message.

Input:
tns:keepAliveRequest
Output:
tns:keepAliveReply
Fault:

openSession

Operation used to start a webservices session with the switch. The credentials of the client and other session-related paramaters are specified as part of the openSessionRequest message. The session details are returned in the openSessionReply message.

NOTE: This operation is used only when the client is connecting using HTTP or HTTPS. This is not needed for clients that are connecting to the switch using telnet or ssh.

Input:
tns:openSessionRequest
Output:
tns:openSessionReply
Fault:

set

Operation used to set data for an object on the switch. The type of object and values to set are specified as part of the setRequest message.
