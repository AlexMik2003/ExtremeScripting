keepAlive

Operation used to keep the webservices session open during periods of inactivity.

The switch can terminate webservices that are idle for long periods of time. The client can specify the idle timeout value as part of the openSession operation. But if there are periods of inactivity that are longer than this timeout, the client can keep the session by sending this keepAliveRequest message.

Input:
tns:keepAliveRequest
Output:
tns:keepAliveReply
Fault:

openSession

Operation used to start a webservices session with the switch. The credentials of the client and other session-related paramaters are specified as part of the openSessionRequest message. The session details are returned in the openSessionReply message.

NOTE: This is used only when client is connecting using HTTP or HTTPS. This is not needed for clients that are connecting to the switch using telnet or ssh.

Input:
tns:openSessionRequest
Output:
tns:openSessionReply
Fault:

runScript

Operation used to run a script when an event occurs on the switch. The script to run and details of the event are specified as part of the runScriptRequest message.

The operation returns an id as part of runScriptReply. This id is used in getExecutionResult to get the results from the script.

Input:
tns:runScriptRequest
Output:
tns:runScriptReply
Fault:

saveConfig

Operation used to save the switch configuration from memory to file.

Input:
tns:saveConfigRequest
Output:
tns:saveConfigReply
Fault:

unbindProfile
