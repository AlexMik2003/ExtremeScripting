getRequest

SuperTypes
SubTypes
Defines the message used for get operations.

The filter is used to specify the set of objects to be returned by the operation. The filter is set by creating a concrete instance of the ExosBase abstract element. A getRequest with a filter instance with no values set will return all the objects that match the instance type. Setting one or more values on the filter and using it in a getRequest will only return objects that match these values.

For example, PortConfig is a concrete instance of ExosBase. This represents port configuration on the switch. Creating an instance of PortConfig and using it in a getRequest will return all the ports configured on the switch. Setting a value on one of the elements of PortConfig, say portList to 1:1, and using it in a getRequest will return port 1:1.

In some cases the number of objects returned can be large. The load on the switch and network traffic can be reduced by using the paging capabilities of getRequest. This can be done specifying a maxSize and setting the action attribute to "next". After a page of data is returned, send the next getRequest using the index from the last object in the page. This is repeated till the response contains 0 rows.

For example, to query for all the VLANs configured on a switch 100 rows at a time the first getRequest will use instance of VlanConfig as filter and set maxSize to 100 and action to "next". The response to this will contain the first 100 VLANs. Since the name is the index for a VLAN, this will be set in the filter which, in this case, is an instance of VlanConfig for the second request. The response will contain the next 100 VLANs. This procedure is repeated till a response is returned that contains 0 rows.
